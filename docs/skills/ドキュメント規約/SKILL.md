# ドキュメント規約

## 契約
- 作成したドキュメントはすべて Obsidian 形式に準拠した Markdown ファイル `docs/*.md` に配置する。
- 必ず先頭に YAML front matter を持つ:
  - `tags`: ドキュメントの分類に使用するタグ。例: `画面`, `機能`
  - `name`: ドキュメントのタイトルや名前
  - `description`: ドキュメントの内容を簡潔に説明するテキスト
- 仕様記述は日本語ベース。既存ドキュメントと同じ言語・粒度で追記する。
