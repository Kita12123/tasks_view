## 概要

この文書は docs/tasks-page.canvas を参照して作成したタスク一覧画面の UI 設計書である。デスクトップはリスト＋詳細ペイン、モバイルは単一カラムでスワイプ操作・無限スクロールを想定する。

## 画面構成

- ヘッダー: ページタイトル（タスク一覧）、グローバルアクション
- ツールバー: 検索バー、タグ絞り込み、期日ソート、タスク追加ボタン
- メイン:
  - 左: タスクリスト（テーブル/カード）
  - 右: 選択タスクの詳細（デスクトップ時）
- フッター: ページネーション（デスクトップ）/無限スクロール（モバイル）
- モバイル: 浮動 + ボタン、スワイプ操作

## コンポーネント詳細

### 検索バー
- プレースホルダー: "タスクを検索"
- 動作: 入力後 Enter でクエリ実行。Esc でクリア。
- API: GET /tasks?q=<keyword>

### タグ（ラベル）絞り込み
- マルチセレクト。選択中はチップ表示。
- API: GET /tasks?tags=tag1,tag2

### 期日ソート
- トグル（昇順/降順）。UIにソート状態のインジケータ表示。
- API: sort=due_date&order=asc|desc

### タスクリスト（行）
- 表示項目: 完了チェック, タイトル, 内容(抜粋, max 2行), 期日(YYYY-MM-DD), 操作(編集/削除)
- タイトルクリック: 右ペインに詳細表示（デスクトップ） / モバイルはタップで詳細画面
- インライン編集: 内容フィールドは編集可能（contenteditable もしくは textarea）。フォーカスアウトまたは Ctrl+Enter で保存（PATCH /tasks/{id}）。編集中は視覚的に示す。
- 完了チェック: 即時 PATCH で completed を切替。成功までローディング表示、失敗時はロールバックとエラートースト。

### 詳細パネル（右ペイン）
- 表示: フルタイトル、フル内容、期日、タグ、操作ボタン（編集、完了切替、削除）
- キーボードアクセス: フォーカス可能、aria-live で更新通知。

### 新規タスクモーダル
- フィールド: タイトル(必須)、内容、期日、タグ
- バリデーション: タイトル必須。保存は POST /tasks。
- モーダル要件: フォーカストラップ、ESCで閉じる、未保存時の確認

### ページネーション / 無限スクロール
- デスクトップ: ページネーション（前へ/次へ、ページ番号）、表示件数セレクタ(10/25/50)
- モバイル: 無限スクロール。下部に読み込みインジケータ表示

## モバイル固有挙動
- タップで詳細表示
- スワイプ右: 完了切替（確認不要）
- スワイプ左: 削除（確認ダイアログ）
- + ボタン: 浮動ボタンで新規作成モーダルを開く

## アクセシビリティ要件
- contenteditable を使う場合は role="textbox" aria-multiline="true" aria-label を付与
- 操作ボタンに aria-label を付与し、キーボードで操作可能にする
- モーダルは aria-modal="true" でフォーカストラップ
- 状態変更（完了/削除など）は aria-live="polite" でスクリーンリーダーに通知
- 色に依存しない状態表示（テキスト + アイコン）

## エラーハンドリング / UX
- ネットワーク失敗はトーストで通知し、必要に応じてロールバック
- 空リストは説明と CTA（タスク追加）を表示
- 編集/保存時は明確な保存中/成功/失敗フィードバック

## API エンドポイント（仕様案）
- GET /tasks?q=&tags=&sort=&order=&page=&per_page=
- POST /tasks {title, content, due_date, tags}
- PATCH /tasks/{id} {title?, content?, due_date?, completed?, tags?}
- DELETE /tasks/{id}

データモデル（例）:
- id: uuid
- title: string
- content: string
- due_date: date|null
- completed: boolean
- tags: string[]

## 実装メモ
- フロント技術: Blazor（既存方針に従いコンポーネント化）
- コンポーネント候補: TaskList, TaskRow, TaskDetail, TaskModal, TaskFilters
- 状態管理: URL クエリでフィルタ/ページを保持（ブックマーク対応）
- パフォーマンス: 長リストは仮想化（virtualization）を導入
- テスト: Playwright による E2E（検索、追加、編集、削除、スワイプ）

## 受け入れ基準
1. 一覧が表示され、検索・タグ絞り込み・期日ソートが機能する
2. タスクの追加・編集・完了・削除が API と連携して動作する
3. デスクトップで詳細ペインが表示され、モバイルでスワイプ/無限スクロールが動作する
4. キーボード操作とスクリーンリーダーへの通知を満たす
